## 🔬 核心发现：选择性锁死与 U 型分布

在长达 5,000 步的训练追踪中，我监控了 **30 层 Transformer** 里每一个 MoE 专家的路由决策。以下是我最重要的发现：

### 1. 选择性锁死 (Selective Deadlock)
并非所有专家都会"死"。在消除了全局锁死后，一种更隐蔽的失效模式出现了：大约 **1/3 的层** 会自发地将 90% 以上的 token 永久分配给同一个专家，且 **单纯增大辅助损失完全无效**。

### 2. U 型分布 (U-Shape Distribution)
通过分析超过 **6500 万 token** 的时序数据，锁死并非随机发生，而是高度集中在 **浅层视觉处理层 (L1-L8)** 和 **深层语义整合层 (L22-L29)**，中间层则保持近乎完美的均衡。

### 3. bfloat16 精度陷阱
一个极易被忽视的数值陷阱：在极小初始化和大数值权重区域，优化器的微小更新量会被硬件截断为 0，导致专家权重 **"纹丝不动"**。
